Fix: XMODEM firmware update - prevent RX thread from consuming ACK bytes

## Problem
Firmware updates via XMODEM were failing at blocks 12-13 with ERROR:46.
Module would reboot due to receiving duplicate blocks after timeouts.

Root cause: Background RX threads in port implementations continuously
call uCxAtClientHandleRx(), which consumes XMODEM ACK bytes (0x06)
before the XMODEM transfer code can read them. This causes:
- 15-second timeouts (U_CX_XMODEM_DEFAULT_TIMEOUT_MS)
- Block retries
- Module interpreting duplicates as errors → reboot with ERROR:46

## Solution
Implemented pause/resume API for RX thread management across all ports:

```c
void uPortAtPauseRx(uCxAtClient_t *pClient);   // Stop RX thread
void uPortAtResumeRx(uCxAtClient_t *pClient);  // Restart RX thread
```

This gives raw binary protocols (XMODEM, etc.) exclusive serial port
access by temporarily disabling AT command parsing.

## Changes

### Core Firmware Update (ucx_api/u_cx_firmware_update.c)
- Added RX thread pause before XMODEM transfer
- Added RX thread resume after XMODEM transfer
- Ensures exclusive serial port access during firmware transfer

### Port Layer Implementations

**Windows (examples/port/u_port_windows.c)**
- Implemented uPortAtPauseRx() using bTerminateRxTask flag
- Implemented uPortAtResumeRx() using CreateThread()
- Tested successfully: all 1352 blocks transferred without errors

**POSIX/Linux (examples/port/u_port_posix.c)**
- Implemented uPortAtPauseRx() using pthread_join()
- Implemented uPortAtResumeRx() using pthread_create()
- Ready for testing on Linux systems

**Zephyr (examples/port/u_port_zephyr.c)**
- Implemented uPortAtPauseRx() using k_work_cancel() + gDisableRxWorker flag
- Implemented uPortAtResumeRx() using k_work_submit()
- Ready for testing on Zephyr-based systems

**FreeRTOS (examples/port/u_port_freertos.c)**
- Added no-op stubs (port uses polling, not background task)
- Included detailed comments for custom implementations using background tasks

**No-OS (examples/port/u_port_no_os.c)**
- Added no-op stubs (no threading, application controls RX timing)
- Already works correctly without modifications

### API Headers (examples/port/u_port.h)
- Added uPortAtPauseRx() function declaration with documentation
- Added uPortAtResumeRx() function declaration with documentation
- Explained critical usage for raw binary protocols

### XMODEM Debug Logging (src/u_cx_at_xmodem.c)
- Enhanced logging to show byte reception timing
- Added detailed block transmission logging
- Improved EOT handling diagnostics

### Firmware Update Tool (examples/firmware_update.c)
- Updated logging messages for clarity
- Maintained compatibility with new pause/resume API
